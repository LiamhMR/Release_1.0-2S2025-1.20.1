11.6. Comandos del Plugin SQL

Dado que ya se han explicado puntos importantes del desarrollo, en los que tenemos la creación y mantenimiento del servidor de Minecraft, la creación de un proyecto utilizando el framework de Maven y el uso de la API de Bukkit para desarrollar un plugin para Minecraft, contamos con un contexto general sobre cómo se construyó el juego y de qué manera se fueron integrando sus distintas modalidades y herramientas de administración.

A continuación se detallan los comandos programados dentro del plugin y su uso práctico. La sección conserva la numeración del informe para que pueda integrarse directamente con el resto de la documentación.

Notación de sintaxis:
  • Los argumentos requeridos se indican entre signos < >.
  • Los argumentos opcionales se indican entre los signos [ ].

11.6.1. Comandos create

/sm create menuzone <nombre>

Requiere haber realizado previamente una selección de dos posiciones con //wand de WorldEdit. Guarda dicha selección con el nombre indicado como una menuZone.

/sm create SQLDUNGEON <mundo>

Define un mundo como escenario de SQL DUNGEON, donde mundo corresponde al nombre del mundo dentro de los archivos del servidor.

/sm create SQLBATTLE [mundo]

Registra un mundo como escenario de SQL Battle. Si el comando se ejecuta sin especificar el mundo, se utiliza el mundo actual del administrador que lo invoca.

/sm create quest <nombre>

Crea un cuestionario QUEST en formato YAML. El plugin genera automáticamente un archivo base dentro del directorio quest/, pensado para que luego se edite y se complete con las preguntas y respuestas correspondientes.

/sm create fixslide <nombre_fixslide> <nombre_zona_slide>

Crea una presentación fija a partir de una menuZone previamente definida como slide. <nombre_zona_slide> debe existir y <nombre_fixslide> será el nombre de la nueva zona de tipo fixslide.

/sm remove <nombre_menu_zone>

Elimina una menuZone. Este comando no se puede deshacer.

/sm createfire <color> [color2] [color3]

Crea un trigger de fuegos artificiales en la posición donde se ejecuta el comando. Funciona como una menuZone de una sola coordenada: al pasar por ese punto, el jugador activará el efecto. El primer color es obligatorio y se pueden añadir hasta dos colores adicionales.

/sm createcreeperfire <color> [color2] [color3]

Su funcionamiento es equivalente al comando anterior, con la diferencia de que el fuego artificial utiliza la forma visual de Creeper.

/sm firework list

Lista todos los triggers de fuegos artificiales actualmente configurados.

/sm firework remove <id>

Permite eliminar un trigger de fuegos artificiales utilizando su identificador. Dicho id puede obtenerse con el comando anterior.

/sm list

Lista todas las menuZones creadas en la configuración del plugin.

11.6.2. Comandos set

Las menuZones son localizaciones o regiones que detectan cuando un jugador se encuentra dentro de ellas y activan un comportamiento específico según el tipo asignado. Los siguientes comandos permiten definir o complementar ese comportamiento.

/sm set menutype <nombre> <tipo>

Permite definir el tipo de comportamiento que tendrá una menuZone. Entre las opciones disponibles se encuentran:

  • chestport: teletransporta al jugador a través de una GUI de decisión.
  • chest: abre una GUI de opciones; esta funcionalidad no quedó terminada en esta entrega.
  • slide: utiliza la zona para mostrar presentaciones gráficas.
  • laboratory: identifica una zona como espacio de prácticas de base de datos.
/sm set requirement <quest_1> <quest_2>

Establece un requisito entre dos quests. Para responder o iniciar <quest_1>, el jugador debe haber completado previamente <quest_2>. Este mecanismo permite construir secuencias de progresión entre actividades.

/sm set requirement <menuzone> <quest> <texto_si_no>

Asocia un quest como requisito de acceso para una menuZone, especialmente en zonas tipo chestport. Si el jugador no ha completado el quest indicado, se mostrará el mensaje configurado y no se le permitirá continuar.

/sm spawnpoint set

Define la posición actual del jugador como el punto de spawn por defecto y configura el mundo actual como lobby principal. Al conectarse al servidor, los jugadores aparecerán en ese punto y las salidas de las modalidades también podrán redirigirlos allí.

11.6.3. Comandos de presentaciones gráficas

Prerequisito: para usar comandos slide, la menuZone correspondiente debe haber sido configurada previamente como tipo slide.
/sm slide <nombre_slide> add <url>

Añade una diapositiva al slide indicado. La URL debe apuntar idealmente a una imagen PNG. Otros formatos como JPG o GIF no fueron completamente validados. La diapositiva se agrega al final de la secuencia existente.

/sm slide <nombre_slide> edit <numero_diapositiva> <url>

Permite sustituir la imagen asociada a una diapositiva ya existente dentro del slide.

/sm slide <nombre_slide> delete <numero_diapositiva>

Elimina una diapositiva específica. Tras la eliminación, las restantes se renumeran automáticamente para mantener continuidad en la secuencia.

Presentaciones fijas (fixslide)

Los fixslide corresponden a una variante de slide en la que la presentación se muestra de forma permanente en una ubicación específica, sin requerir que el jugador active una zona con su movimiento. Para crearlos, primero se debe disponer de un slide convencional con sus diapositivas configuradas.

/sm disabled <nombre>

Desactiva una menuZone. Se utiliza, por ejemplo, cuando una zona slide deja de requerir activación directa porque será reutilizada como fixslide.

/sm fixslide <nombre_fixslide> posfix

Guarda la posición fija en la que se renderizará la presentación.

/sm fixslide <nombre_fixslide> fix <+X|-X|+Z|-Z>

Configura la orientación de la presentación, es decir, la dirección hacia la que mirarán las diapositivas dentro del mundo.

/sm fixslide <nombre_fixslide> <backbutton|nextbutton>

Posiciona el botón interactivo para retroceder o avanzar de diapositiva en la ubicación donde se ejecuta el comando.

/sm fixslide list

Muestra la lista de todos los fixslide registrados.

11.6.4. Comandos de Harry

Harry es un NPC que cumple una función orientadora dentro del servidor. Cuando el jugador interactúa con él, puede recibir indicaciones sobre el espacio en el que se encuentra o sobre las acciones que puede realizar.

/sm newharry <nombre>

Crea un Harry en la posición exacta donde se ejecuta el comando.

/sm harry <nombre> addLine <texto>

Añade una línea de diálogo al Harry especificado. El texto puede incluir códigos de color utilizando el prefijo &, por ejemplo &6Hola Mundo para mostrar el mensaje en amarillo.

/sm harry <nombre> editLine <número_linea> <texto>

Permite editar una línea específica del diálogo existente, indicada por su número dentro del conjunto de líneas del NPC.

/sm harry <nombre> lines

Muestra todas las líneas de diálogo actualmente configuradas para ese Harry.

/sm harry <nombre> remove

Elimina por completo al NPC indicado.

/sm harry <nombre> reset

Es un comando utilitario pensado para resolver duplicaciones de entidades. Al reiniciar el servidor, puede ocurrir que algunas entidades de Harry no sean limpiadas correctamente si la zona no se encontraba cargada. Este comando debe ejecutarse cerca del NPC para limpiar duplicados y volver a posicionarlo.

11.6.5. Comandos SQL

/sm sql schema

Permite visualizar por chat la estructura de tablas de la base de datos utilizada para SQL Dungeon. Fue un comando provisional incorporado cuando todavía no se había formalizado el diagrama ERD.

/sm sql here lvl add <número> <dificultad>

Agrega un nivel al mundo SQL DUNGEON actual. Número corresponde al identificador del nivel y dificultad es un valor entre 1 y 5, utilizado para seleccionar el grupo de desafíos SQL adecuado desde el banco de consultas.

/sm sql here lvl delete <número>

Elimina un nivel específico dentro del mundo SQL DUNGEON actual.

/sm sql here set entry <número_nivel>

Configura la posición actual como punto de interacción del nivel indicado. En esa ubicación el jugador podrá introducir la consulta SQL que resuelve el desafío. En la implementación del proyecto, habitualmente se usaron bloques de comando para marcar visualmente este punto.

/sm sql info

Detalla el progreso del jugador dentro del SQL DUNGEON actual, el estado de la sesión y la información de los niveles configurados.

/sm sql bank info

Muestra información general sobre el banco de desafíos SQL, incluyendo la cantidad disponible por nivel de dificultad.

/sm sql bank regenerate <mundo_SQL_DUNGEON> <nivel>

Estado: deprecado en la práctica. Originalmente se usó para regenerar el challenge asignado a un nivel, aunque después el diseño evolucionó hacia una selección más dinámica de desafíos.

/sm sql repair

Estado: comando de reparación heredado. Se empleó para corregir niveles que hubiesen quedado sin challenge asociado, pero en la versión actual su uso es más bien residual.

11.6.6. Comandos SQL Battle

SQL Battle es una modalidad de defensa de castillo en la que los jugadores deben prepararse e interactuar con una base de datos SQL para desplegar recursos antes de cada oleada. Sus comandos cubren tanto la configuración espacial del mundo como la administración y depuración de sesiones.

Registro y configuración del mundo

/sm sqlbattle here set entry

Guarda, a partir de una selección de WorldEdit, la zona en la que se capturará el chat SQL del jugador. Dentro de esta región se interceptan los mensajes para tratarlos como consultas SQL.

Prerequisito: requiere una selección activa con //wand.
/sm sqlbattle here set wavestart

Guarda la posición actual como punto de inicio de oleada. Allí serán ubicados los jugadores participantes cuando la fase activa comience.

/sm sqlbattle here set checkpoint

Define la posición actual como checkpoint de respawn para pruebas o reaparición durante la modalidad.

/sm sqlbattle here set prewave

Establece la zona de preparación previa a la oleada, donde los jugadores disponen de un momento para consultar la base de datos y planificar la defensa.

/sm sqlbattle here set summonzone

Guarda una región seleccionada con WorldEdit como zona de invocación. Allí se materializan los resultados de ciertas acciones o invocaciones ligadas a la fase SQL.

Prerequisito: requiere una selección activa con //wand.
/sm sqlbattle here set enemyspawn

Define, a partir de una región de WorldEdit, el espacio desde el cual aparecerán los enemigos en cada oleada.

Prerequisito: requiere una selección activa con //wand.
/sm sqlbattle here set castle

Configura la región del castillo que los jugadores deben defender. La modalidad evalúa esta zona como el objetivo principal del asedio.

Prerequisito: requiere una selección activa con //wand.
/sm sqlbattle here status

Muestra el estado resumido del SQL Battle configurado en el mundo actual: si está activo, si la configuración está completa y qué zonas fundamentales han sido registradas.

/sm sqlbattle here schema

Muestra por chat un resumen del esquema de tablas utilizado en la base de datos en memoria de SQL Battle.

Inicio, detención y control

/sm sqlbattle start [mundo]

Inicia SQL Battle. Si se ejecuta desde un jugador sin indicar mundo, intenta comenzar la sesión en el mundo actual. Si se ejecuta indicando un mundo, registra a los jugadores de esa arena e inicia el flujo de la modalidad.

/sm sqlbattle stop [mundo]

Detiene la sesión de SQL Battle y coloca el mundo en estado pacífico, cerrando además las sesiones activas de preparación.

/sm sqlbattle difficulty <peaceful|easy|normal|hard> [mundo]

Cambia la dificultad del mundo asociado a la arena SQL Battle. Se utiliza principalmente para depuración o preparación del escenario.

/sm sqlbattle reset [mundo]

Resetea el estado de prueba de los jugadores. Si lo ejecuta un jugador dentro de una arena SQL Battle, aplica el reinicio sobre sí mismo; si se especifica un mundo, puede resetear a los jugadores de esa arena.

/sm sqlbattle forcestage <1|2|3> [jugador]

Fuerza manualmente una etapa interna de la oleada para fines de testeo. Es un comando administrativo pensado para validar transiciones de estados.

/sm sqlbattle respawncheckpoint [jugador]

Envía al jugador indicado, o al propio ejecutor si no se especifica, al checkpoint configurado para la arena actual.

/sm sqlbattle stopgamemode

Pausa el forzado automático del modo aventura dentro de SQL Battle, permitiendo editar temporalmente la arena con mayor libertad.

/sm sqlbattle continuegamemode

Reanuda el forzado del modo aventura una vez finalizada la edición o depuración del escenario.

Inspección, apoyo y depuración

/sm sqlbattle get <item> [jugador]

Entrega un item de depuración relacionado con SQL Battle al jugador indicado. Se trata de una herramienta administrativa para pruebas del modo de juego.

/sm sqlbattle suggest <1|2|3|4>

Ejecuta una sugerencia de preparación predefinida durante la fase SQL. El sistema usa identificadores numéricos para disparar recetas o acciones guiadas.

/sm sqlbattle debug [mundo]

Muestra información visual de depuración, como partículas o indicadores espaciales, para ayudar a verificar la configuración de la arena.

/sm sqlbattle list

Lista todos los mundos registrados como SQL Battle e indica si su configuración está completa o todavía incompleta.

/sm sqlbattle info [mundo]

Muestra la configuración detallada de una arena SQL Battle: dificultad, zonas registradas, puntos de entrada, checkpoint, preparación, invocación, spawn enemigo y castillo.

/sm sqlbattle ranking [top]

Muestra el ranking global de puntos acumulados en SQL Battle, limitado por defecto a los 10 primeros jugadores, aunque puede ajustarse el número de resultados.

/sm sqlbattle remove [mundo]

Elimina la configuración SQL Battle del mundo indicado, o del mundo actual si el comando lo ejecuta un jugador dentro de él.

11.6.7. Comandos QUEST y encuestas

El sistema QUEST se utiliza para integrar cuestionarios y requisitos de progresión dentro del servidor. Se complementa con el sistema de encuestas, utilizado especialmente para pre-test, post-test y otras actividades de evaluación.

Comandos QUEST

/sm create quest <nombre>

Crea un archivo base de quest con estructura YAML. Sirve como punto de partida para preparar un cuestionario o actividad evaluativa dentro del juego.

/sm start quest <nombre>

Inicia el quest indicado para el jugador que ejecuta el comando. Su uso está pensado para lanzar actividades concretas una vez que el archivo ya fue configurado.

/sm set requirement <quest_1> <quest_2>

Define prerequisitos entre quests, de manera que uno no pueda responderse o activarse sin haber completado otro previamente.

/sm set requirement <menuzone> <quest> <texto_si_no>

Vincula un quest a una zona del mundo para restringir el acceso hasta que el requisito académico haya sido cumplido.

Comandos de encuestas

/sm survey create <nombre_encuesta>

Crea una nueva encuesta vacía. A partir de ella se podrán agregar preguntas que luego serán respondidas por los jugadores.

/sm survey add <nombre_encuesta> <pregunta>

Añade una pregunta a la encuesta especificada. Las preguntas se almacenan en orden y pueden ser usadas posteriormente como parte de un formulario in-game.

/sm survey edit <nombre_encuesta> <nro_pregunta> <nueva_pregunta>

Edita una pregunta ya existente dentro de la encuesta, indicando el número correspondiente y el nuevo texto.

/sm survey start <nombre_encuesta>

Inicia la encuesta indicada para el jugador que ejecuta el comando.

/sm survey list

Lista todas las encuestas disponibles, mostrando además cuántas preguntas y respuestas registra cada una.

/sm survey info <nombre_encuesta>

Muestra información detallada de la encuesta: cantidad de preguntas, total de respuestas y listado de ítems configurados.

/sm survey stats <nombre_encuesta>

Presenta las estadísticas acumuladas de una encuesta, incluyendo promedios y distribución de calificaciones por pregunta.

/sm defaultsurvey <nombre_encuesta>

Establece una encuesta como post-test por defecto en la posición actual del jugador. Esto permite que, al finalizar cierta actividad, el servidor disponga de un mecanismo estándar para activar la evaluación final.
